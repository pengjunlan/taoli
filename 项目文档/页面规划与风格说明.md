# 多交易所套利系统页面规划与风格说明

## 1. 文档目的

本文件用于整理当前项目第一阶段的页面设计方向。现阶段先不编写核心套利逻辑，优先完成前端页面骨架与信息结构设计，为后续接入跨交易所合约价差套利、跨交易所资金费套利做准备。

系统定位为：

- 多交易所套利系统
- 重点支持跨交易所合约价差套利
- 重点支持跨交易所资金费套利
- 第一阶段先完成页面展示、配置、监控外壳

## 2. 页面风格方向

参考风格采用偏 `CoinGlass` 的数据站样式，而不是传统重后台风格。

整体设计建议如下：

- 采用浅色系界面，白底 + 浅灰分区 + 深色文字
- 顶部增加一条全局指标栏，展示市场与系统核心数据
- 主导航以横向导航为主，不优先使用厚重的左侧后台菜单
- 页面主体以大表格、筛选栏、标签切换为核心
- 图表作为辅助信息，表格和机会列表作为主视觉中心
- 颜色只承担状态表达，不做过度装饰

状态颜色建议：

- 绿色：做多、正收益、低风险
- 红色：做空、负收益、高风险
- 蓝色：链接、交互操作、次级强调
- 灰色：说明信息、弱提示、辅助分隔

## 3. 整体导航建议

建议第一版使用以下导航结构：

- 顶部指标栏：展示市场和系统关键指标
- 顶部主导航：切换核心页面
- 页面内部使用二级标签和筛选栏

建议主导航栏目：

- 首页
- 资金费套利
- 价差套利
- 策略管理
- 持仓订单
- 账户配置
- 风控告警

右上角建议保留：

- 搜索框
- 用户菜单
- 登录/退出入口

## 4. 第一阶段核心页面清单

第一阶段建议优先完成以下 8 个页面。

### 4.1 登录页

用途：

- 用户进入系统的统一入口

功能清单：

- 账号密码登录
- 注册
- 记住登录状态
- 退出登录
- 基础安全提示

页面风格建议：

- 保持简洁
- 不使用复杂插画
- 登录成功后跳转首页

### 4.2 首页总览页

用途：

- 作为系统总入口，集中展示市场、策略、收益、风险状态

功能清单：

- 顶部全局指标栏
- 当前运行策略数
- 当前连接交易所数
- 今日预估收益
- 当前最大资金费套利机会
- 当前最大价差套利机会
- 最近告警
- 最近成交
- 热门套利机会表格
- 快捷入口：新建策略、账户配置、风控设置

页面风格建议：

- 首页要尽量贴近参考图的视觉感觉
- 以卡片 + 表格为主
- 图表只作为辅助，不喧宾夺主

### 4.3 资金费套利页

用途：

- 展示跨交易所资金费差套利机会
- 支持快速筛选、观察和后续策略创建

功能清单：

- 页面主标签：资金费差套利
- 子标签：全部机会、自选、已创建策略
- 筛选交易所
- 筛选币种
- 筛选最小年化
- 筛选最小净资金费率
- 筛选距离结算时间
- 搜索交易对
- 表格排序
- 快速加入自选
- 快速创建策略

主表格字段建议：

- 排名
- 币种
- 交易所组合
- 做多/做空方向
- 当前年化
- 净资金费率
- 价差率
- 深度/流动性
- 距离结算
- 操作

### 4.4 合约价差套利页

用途：

- 展示跨交易所合约价差机会
- 支持筛选最有价值的跨所价差机会

功能清单：

- 页面主标签：跨交易所价差套利
- 筛选交易所组合
- 筛选币种
- 筛选最小价差
- 筛选手续费后净收益
- 筛选盘口深度
- 搜索交易对
- 表格排序
- 快速加入自选
- 快速创建策略

主表格字段建议：

- 排名
- 币种
- 多头交易所
- 空头交易所
- 最新价差
- 净价差
- 手续费成本
- 可成交深度
- 建议仓位
- 操作

### 4.5 策略管理页

用途：

- 展示用户已创建的套利策略
- 作为策略运行管理中心

功能清单：

- 策略列表
- 按策略类型筛选
- 按状态筛选
- 查看策略名称
- 查看交易对
- 查看交易所组合
- 查看当前收益
- 查看当前仓位
- 查看风险状态
- 启动策略
- 暂停策略
- 停止策略
- 进入策略详情

策略类型建议：

- 资金费套利
- 价差套利

策略状态建议：

- 运行中
- 暂停
- 异常

### 4.6 持仓与订单页

用途：

- 集中查看当前仓位、未成交订单、历史成交及异常情况

功能清单：

- 当前持仓列表
- 按交易所查看多空仓位
- 按策略查看对冲是否完整
- 当前未成交订单
- 历史订单
- 历史成交
- 手动平仓
- 手动撤单
- 异常订单提示

页面重点：

- 强调对冲完整性
- 强调异常状态
- 便于快速人工接管

### 4.7 账户与交易所配置页

用途：

- 配置多个交易所账户
- 为后续策略运行提供账户和权限基础

功能清单：

- 添加交易所 API
- 支持多个交易所账户
- 选择交易所类型
- 填写 API Key / Secret / Passphrase
- 测试连接
- 查看连接状态
- 查看余额
- 查看保证金模式
- 设置杠杆
- 配置交易对白名单
- 启用/停用账户
- 删除账户

建议支持的交易所类型：

- Binance
- OKX
- Bybit
- Gate

### 4.8 风控与告警页

用途：

- 对系统整体风险进行统一配置与监控

功能清单：

- 最大单策略仓位限制
- 最大总仓位限制
- 最大滑点阈值
- 最大未对冲时长
- API 断连处理策略
- 下单失败处理策略
- 全局停机按钮
- 告警历史
- 告警等级筛选
- 按策略筛选告警
- 按交易所筛选告警

页面重点：

- 风控配置要直观
- 风险状态要醒目
- 必须支持一键停机

## 5. 最优先统一风格的页面

以下 3 个页面最适合优先采用参考图的风格做统一设计：

- 首页总览页
- 资金费套利页
- 合约价差套利页

原因：

- 这 3 个页面最依赖大表格和筛选交互
- 最能体现“数据站式套利系统”的产品气质
- 最适合复用统一的页面骨架

建议统一页面骨架：

- 顶部市场/系统指标条
- 横向主导航
- 页面标题
- 标签切换
- 筛选栏
- 大表格
- 右上角搜索和快捷操作

## 6. 第一阶段实现建议

如果当前目标是先快速完成页面壳子，建议开发顺序如下：

1. 登录页
2. 首页总览页
3. 资金费套利页
4. 合约价差套利页
5. 策略管理页
6. 持仓与订单页
7. 账户与交易所配置页
8. 风控与告警页

这个顺序的好处：

- 先把整体视觉风格定下来
- 先把最核心的数据页做出来
- 后续再逐步补充策略逻辑和真实接口时，页面承接关系会更顺

## 7. 当前结论

当前项目不应优先从登录或普通后台表单思路出发，而应优先从“套利机会展示 + 策略监控”出发。

页面层面建议以以下三类为核心：

- 机会发现页面
- 策略管理页面
- 风控监控页面

其中，最重要的视觉风格基准页为：

- 首页总览页
- 资金费套利页
- 合约价差套利页
