# 多交易所套利系统页面规划与风格说明

## 1. 文档目的

本文件用于说明当前版本页面规划、视觉方向和信息结构原则。

当前项目已经从最早的“原型规划阶段”进入“页面结构基本稳定阶段”，因此本文件以现有页面实现为准，不再保留已经取消的设计设想。

系统当前定位为：

- 多交易所套利系统
- 支持资金费套利
- 支持跨交易所价差套利
- 以自动扫描、统一规则执行、账户调度和风控追踪为核心
- 支持多用户同时使用，且用户数据完全隔离


## 2. 当前页面风格方向

当前页面已经确定为偏数据面板风格，而不是传统厚重后台风格。

整体方向如下：

- 采用浅色界面
- 以白底、浅灰分区、深色文字为主
- 页面主体以卡片和大表格为核心
- 使用顶部横向导航，不采用左侧复杂菜单
- 颜色主要承担状态表达，不做过度装饰
- 页面尽量减少没有实际功能价值的按钮和说明块

状态颜色约定：

- 绿色：正收益、做多、低风险、正常状态
- 红色：负收益、做空、高风险、异常状态
- 蓝色：主强调、可点击入口、系统主色
- 灰色：辅助说明、弱提示、非核心信息


## 3. 当前导航结构

当前导航结构已经固定为顶部主导航，栏目如下：

- 首页
- 资金费套利
- 价差套利
- 规则管理
- 持仓订单
- 账户调度
- 风控告警

导航辅助区当前仅保留：

- 退出登录

以下旧规划内容已取消：

- 顶部搜索框
- 用户中心按钮
- 顶部指标栏


## 4. 当前页面结构原则

### 4.1 首页

- 首页只承担概览和预览职责
- 首页不承担完整数据页职责
- 首页核心是指标卡片 + 混合机会预览表
- 首页列表只展示少量预览数据，并通过“更多”跳转到完整页

### 4.2 机会页

- 资金费套利页和价差套利页采用统一表格结构
- 差异通过字段含义体现，而不是完全不同布局
- 资金费页保留“距离结算”
- 价差页保留“建议仓位”
- 两页都展示对冲后的双边持仓字段

### 4.3 规则页

- 规则页展示的是全局规则组
- 不是单个交易对策略表
- 页面更强调“执行逻辑说明”，而不是到处放手动操作按钮

### 4.4 持仓订单页

- 强调执行结果查看
- 强调持仓完整性、异常订单和最近成交
- 所有列表采用统一表格表达

### 4.5 账户调度页

- 账户页是系统基础页之一
- 不只是 API 录入页
- 资金分布、调拨记录、接收地址配置、账户状态统一放在这一页

### 4.6 风控页

- 风控页强调风险解释能力
- 不只是规则阈值展示
- 需要能说明哪些风险事件影响了自动执行和收益稳定性

### 4.7 多用户原则

- 当前所有页面都要默认服务于多用户场景
- 页面上展示的账户、机会、规则、持仓、订单、调拨和风控数据，未来都必须只属于当前登录用户
- 不能出现“全局共用一套账户或规则数据”的实现思路
- 后续如果加入管理员能力，也必须与普通用户视角严格区分


## 5. 当前通用设计规则

- 页面头部尽量简洁，不堆无实际功能的入口
- 大部分核心信息通过卡片和表格表达
- 列表页面统一接入分页
- 概览页列表只展示少量预览数据
- 表格优先展示业务字段，不优先展示操作列
- 已删除的模块不再回加，除非后续有明确真实功能支撑
- 做功能时默认不修改已确认页面
- 页面结构、布局、文案、视觉改动都需要单独确认后再执行


## 6. 当前已经取消的旧规划内容

以下内容属于早期规划，但当前版本已经不再采用：

- 观察按钮
- 导出按钮
- 大量筛选区和搜索区
- 顶部搜索框
- 顶部用户中心按钮
- 首页收益曲线模块
- 以“演示 / 原型 / 开发中”为主的提示文案
- 围绕单个交易对手工创建套利任务的设计思路


## 7. 当前最重要的三个基准页面

当前最适合作为统一风格基准的页面为：

- 首页总览页
- 资金费套利页
- 价差套利页

原因：

- 这三页最能体现当前系统主链路
- 这三页的表格结构已经基本统一
- 后续其他页面继续优化时，应尽量向这三页的简洁程度靠拢


## 8. 后续页面优化建议

当前页面主体已经稳定，后续如果继续优化，建议优先做这几类收尾：

- 继续压缩账户调度页的信息密度
- 继续统一规则页、持仓页、风控页的表格视觉节奏
- 将后端真实数据接入后，再决定是否恢复局部筛选能力
- 只有在真实业务需要时，再增加可执行按钮或详情页入口
- 接入真实后端时，优先把多用户认证和数据隔离打牢


## 9. 当前结论

当前页面风格已经从“原型堆叠”调整为“结构统一、信息直接、尽量少废模块”的方向。

后续继续补功能时，建议遵循以下顺序：

1. 接真实数据
2. 接真实执行逻辑
3. 接真实配置能力
4. 最后再决定是否增加更复杂交互
