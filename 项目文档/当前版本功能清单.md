# 多交易所套利系统当前版本功能清单

## 1. 文档说明

本清单基于当前已经完成的页面结构整理，不再沿用最早的页面原型规划。

当前版本重点不是“单个交易对手工建策略”，而是：

- 配置交易所账户后，系统自动监控资金费套利机会
- 配置交易所账户后，系统自动监控跨所价差套利机会
- 命中全局规则参数后，系统按规则执行套利
- 账户资金调度、自动均衡、接收地址配置作为核心基础能力
- 风控、持仓、订单、风险事件围绕自动套利主链路展开


## 2. 当前页面总览

当前页面共 9 个：

- 登录页 `/login`
- 注册页 `/register`
- 首页 `/dashboard`
- 资金费套利页 `/funding-arbitrage`
- 价差套利页 `/spread-arbitrage`
- 自动规则管理页 `/strategies`
- 持仓与订单页 `/positions-orders`
- 账户与资金调度页 `/accounts`
- 风控与告警页 `/risk-alerts`


## 3. 产品主逻辑

### 3.1 套利机会发现

- 系统监控范围不是手工逐个交易对创建
- 只要交易所账户已配置并纳入系统，就进入统一监控范围
- 系统自动扫描两类机会：
- 资金费套利
- 跨交易所价差套利
- 机会命中规则后进入执行队列

### 3.2 规则驱动执行

- 规则是全局规则组，不是某一个币种的临时开关
- 规则组按套利类型划分
- 规则组决定：
- 扫描范围
- 最低收益门槛
- 执行条件
- 仓位规模
- 风险限制

### 3.3 资金调度优先级提升

- 账户页不只是账户录入页
- 资金平均分配、自动失衡修复、调拨记录、接收地址配置都属于核心能力
- 当资金不足或分布失衡时，需要支持：
- 一键平均分配
- 自动均衡
- 调拨任务跟踪
- 接收地址配置与校验

### 3.4 风控与收益联动

- 风控影响自动开仓
- 风控影响资金调度
- 风控影响收益曲线稳定性
- 风险事件需要能追溯到策略、账户、连接、订单等对象

- 标题采用当前代码中的正式标题口径

- 最简注册入口

当前页面功能：

- 密码输入

### 4.3 首页总览页

当前定位：

- 系统主入口
- 用于统一查看两类套利机会、关键指标和风险状态

当前页面模块：

- 顶部页面标题与入口按钮

- 机会筛选区

当前页面功能：

- 展示已连接交易所数量
- 展示风险事件数量

- 首页列表同时展示：
- 资金费套利
- 价差套利
- 首页列表按类型区分显示
- 首页列表只展示前几条预览数据
- 分别跳转到资金费完整页和价差完整页

当前页面字段原则：

- 首页不能只按资金费套利字段展示
- 两类套利要有类型区分
- 资金费套利可展示“距离结算”
- 价差套利不可使用“距离结算”
- 首页混合列表必须使用通用字段结构

当前分页规则：

- 列表页已接入分页组件
- 概览页中的列表以预览展示为主
- 更多按钮用于进入完整数据页


## 6. 当前版本需要保持一致的设计原则

- 所有主列表页面尽量统一表格结构与操作区布局
- 首页只做预览，不承担完整列表职责
- 两类套利必须区分展示，不混用错误字段
- 不再保留“观察”相关无效按钮
- 不再保留“导出”这类当前无实际作用按钮
- 不再保留“演示模式”“原型说明”等提示文案
- 所有操作入口命名尽量使用正式业务语言

## 8. 当前结论

当前版本页面核心已经明确为四条主线：

- 套利机会发现
- 自动规则执行
- 账户资金调度
- 风控与结果追踪

后续继续补功能时，建议都围绕这四条主线推进，不再回到“传统后台配置页优先”的思路。
