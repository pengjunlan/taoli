多交易所套利系统文件结构说明

一、页面实现推荐技术

1. 后端框架：FastAPI
2. 页面模板：Jinja2
3. 前端交互：HTML + CSS + 原生 JavaScript ES Modules
4. 实时数据：WebSocket
5. 图表展示：ECharts
6. 大表格展示：Tabulator 或原生表格增强
7. 数据库：SQLite

说明：
1. 第一阶段先完成页面骨架，不急着做复杂前后端分离
2. 页面、接口、策略、交易所接入统一放在 app 目录中
3. 页面层负责展示和交互，不直接承载交易逻辑

三、页面文件存放规范

1. 每个页面单独一套文件
   templates/pages/xxx.html
   static/css/pages/xxx.css
   static/js/pages/xxx.js

2. 公共布局统一放在 base
   包括总布局、顶部指标栏、主导航栏

3. 可复用模块统一放在 components
   包括卡片、表格、筛选栏、弹窗、告警列表

4. 页面脚本只负责：
   页面渲染
   页面交互
   调用接口
   订阅实时数据

5. 交易和策略逻辑不要写进页面 js
   页面层只做展示和操作入口

五、当前结构设计原则

1. 所有代码只写在 app 目录内
2. 页面结构、样式、交互分离
3. 公共组件优先抽离
4. 页面先做骨架，后接真实数据
5. 先支持多交易所展示，再逐步接入真实套利逻辑
